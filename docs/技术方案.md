# GoNacos 技术方案

## 目标

新建 `gonacos` 独立 Go 项目，完整复刻 Nacos 当前 v3 服务端能力：

- 支持当前最新 v3 HTTP API、gRPC SDK 协议、鉴权协议、Admin API、Console API。
- 支持 Nacos Web 控制台完整功能，包括配置、注册发现、命名空间、集群、插件、用户/角色/权限、AI 资源等页面。
- 基于 `nacos-sdk-go` 做客户端兼容性验收。
- 参考 `other/nacos` 的业务行为和测试矩阵，但不翻译 Java 包结构。
- 不兼容老协议：不实现 `/v1/**`、`/v2/**`，不实现 pre-3.0 存储迁移逻辑。

## 协议范围

HTTP 只实现当前 v3：

- `/v3/client/**`：配置查询、注册发现实例注册/列表/注销、AI prompt/skill/AgentSpec 客户端读取。
- `/v3/console/**`：Web 控制台完整后端接口。
- `/v3/admin/**`：运维、管控、集群、配置、注册发现、AI、鉴权等 Admin API。
- `/v3/auth/**`：默认 Nacos 鉴权插件的登录、用户、角色、权限接口。

gRPC 实现 `nacos-sdk-go` 当前 proto：

- `Payload{metadata, body}`
- `Request.request`
- `RequestStream.requestStream`
- `BiRequestStream.requestBiStream`

服务端按 `Metadata.type` 和 `Any` body 类型分发，只接受当前 SDK 使用的请求类型，未知或老协议请求返回 Nacos 兼容错误。

当前已完成的协议基础：

- 已固化官方 client/admin/console Swagger 到 `api/openapi/upstream/`。
- 已固化 `nacos-sdk-go` gRPC proto 到 `api/proto/nacos_grpc_service.proto`。
- 已生成 `api/openapi/manifest.json`，当前包含 297 个 HTTP operation、3 个 gRPC service/rpc。
- 已实现 `cmd/gonacos-contract` 的 `contract-generate` 和 `contract-verify`。
- HTTP server 已根据 manifest 注册所有 v3 路由；健康/状态接口已返回成功。
- 已实现 console/admin 命名空间 CRUD、list、exist/check 的内存版纵切面，默认内置 `public` 命名空间。
- 已实现配置中心核心纵切面：admin/console 发布、查询、删除、列表、批量删除、克隆、历史列表/详情/上一版本/命名空间配置列表、zip 导入/导出、监听状态查询、beta 发布/查询/停止、admin 元数据更新，以及 client 配置查询，返回 md5、lastModified 和 contentType。
- 其它接口当前返回标准 501 stub，后续按模块替换为真实实现。

## 核心模块

### 配置中心

- 配置发布、查询、删除、更新、CAS、md5、类型、描述、标签、灰度/beta。
- 历史版本、上一版本、快照、回滚。
- 批量删除、导入 zip、导出 zip、克隆。
- 配置监听、按 IP 查询监听状态、gRPC/长轮询通知。

### 注册发现

- 服务 CRUD、服务列表、selector 类型、subscriber 查询。
- 集群元数据、健康检查元数据。
- 实例注册、更新、查询、注销、权重、健康、启用状态、metadata、ephemeral。
- 心跳租约、过期剔除、服务订阅推送。

### 鉴权

- 默认 `nacos` 用户名密码登录和 token 签发。
- 用户、角色、权限 CRUD，管理员初始化，密码修改。
- 权限资源遵循当前规范：
  `{namespaceId}:{group}:{signType}/{resourceName}`，
  控制台资源使用 `console/users`、`console/roles`、`console/permissions`。
- 预留 RAM/OIDC 插件接口，但第一阶段完整实现默认鉴权。

### AI 注册中心

- MCP server、tool 导入校验和执行。
- A2A agent 生命周期和版本。
- Prompt draft/submit/publish/force-publish/redraft/online/offline/label/description/bizTags/download。
- Skill draft/fork/upload/batch upload/publish/scope/online/offline/download。
- AgentSpec draft/upload/resource tree/version/label/scope/online/offline。
- AI resource import source/search/validate/execute。
- Pipeline 和 Copilot API 保持兼容响应；外部 LLM 调用做可插拔实现，测试默认关闭。

### Web 控制台

以 `../other/nacos/console-ui-next/src/pages` 为完整功能清单，至少覆盖：

- 登录、注册/管理员初始化、欢迎页。
- 命名空间、集群管理。
- 配置管理、详情、编辑、历史、回滚、同步、监听查询。
- 服务管理、服务详情、实例和 subscriber。
- 插件管理。
- 用户、角色、权限管理。
- 设置中心。
- MCP、A2A Agent、Prompt、Skill、AgentSpec、AI 资源导入、上传、详情、版本工作流。

Web 前端只依赖 `/v3/console/**` 和 `/v3/auth/**`，由 GoNacos 静态服务托管。

## 存储与集群

- 单机模式：SQLite 或 Pebble 类嵌入式存储，便于开发和测试。
- 生产模式：PostgreSQL/MySQL 兼容 SQL 存储。
- 集群模式：按领域划分 raft group，覆盖配置、注册发现、鉴权、AI 资源、成员信息。
- ephemeral 实例以内存状态为主，通过心跳租约和 raft 事件复制；persistent 数据必须可重启恢复。

## 非目标

- 不支持 `/v1/**`、`/v2/**`。
- 不支持 pre-3.0 旧表迁移。
- 不兼容 Java SPI 二进制插件。
- 不逐字复刻 Java 内部类结构。
- 不内置外部 LLM、MSE、云 RAM、OIDC 服务，只保留可插拔接口和兼容响应。
