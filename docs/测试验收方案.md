# GoNacos 测试验收方案

## 验收原则

GoNacos 的验收标准是“当前 Nacos v3 客户端和 Web 控制台无需改代码即可使用”。所有可观察行为以当前 v3 OpenAPI、gRPC proto、`other/nacos` 测试矩阵和 `nacos-sdk-go` 为准。

## 测试层级

- 单元测试：参数校验、错误映射、权限判断、md5/CAS、租约过期、导入导出解析、AI 资源状态机。
- Handler 测试：用 `httptest` 覆盖每个 `/v3/**` endpoint。
- SDK 兼容测试：直接用 `nacos-sdk-go` 连接 GoNacos。
- 契约测试：同一请求语料分别打到 GoNacos 和参考 Nacos，比较状态码、响应 envelope、字段、错误码。
- 浏览器测试：Playwright 覆盖 Web 控制台主流程。
- 集群测试：三节点进程、重启、分区、快照、leader 切换。
- 性能和 soak：吞吐、延迟、内存、goroutine 泄漏、长连接监听稳定性。

## SDK 兼容验收

新增 `tests/sdkcompat`，直接依赖 `github.com/nacos-group/nacos-sdk-go/v2`。

必测流程：

- Config：publish、get、delete、search、listen、cancel listen、CAS、服务重启后的本地缓存行为。
- Naming：register、batch register、list/select、subscribe、unsubscribe、deregister、heartbeat 过期。
- Security：用户名密码 token、鉴权 header、无权限拒绝。
- TLS：HTTP 和 gRPC 自定义 CA。
- 错误：缺必填字段、未知 namespace、非法 group、非法 weight/port、未授权。

## Web 控制台验收

Playwright 必测：

- 登录/登出、修改密码、管理员初始化。
- 命名空间 CRUD 和跨页面 namespace 切换。
- 配置新建、编辑、发布、历史、diff、回滚、导入、导出、克隆。
- 服务创建、编辑、详情、实例更新/删除、cluster 元数据、subscriber。
- 用户、角色、权限 CRUD 和权限生效。
- 插件列表、详情、配置、启停状态。
- MCP、A2A、Prompt、Skill、AgentSpec、上传、下载、版本、上下线、导入。
- 桌面和移动宽度冒烟测试。

每个 UI 测试必须同时断言界面状态和后端 API 状态。

## 集群验收

三节点 GoNacos：

- 节点发现和 member list 收敛。
- 任一节点写入配置，其它节点可读。
- Naming 心跳、过期、订阅推送在节点间一致。
- 鉴权和权限变更复制后立即生效。
- AI 资源发布、下载在 leader 切换后仍可用。
- kill leader、少数派网络分区、恢复分区后无已提交写丢失。
- 快照恢复、备份恢复后 API 可见状态一致。

## 性能门槛

初始门槛，后续基于基准测试调整：

- 配置查询 1,000 RPS 下 p95 < 20 ms。
- 10,000 实例服务列表 p95 < 30 ms。
- 10,000 配置监听长连接内存稳定，无 goroutine 泄漏。
- 50,000 ephemeral 实例稳态心跳处理低于 1 个 CPU core。
- Web 列表类 API 真实分页下 p95 < 100 ms。

## 当前已建立的自动门禁

- `make contract-generate`：从已固化 OpenAPI/proto 重新生成 manifest。
- `make contract-verify`：校验 manifest 未漂移。
- `GOWORK=off go test ./...`：当前覆盖契约解析、健康接口、manifest 路由 stub、未知路由、命名空间 console/admin 生命周期和校验行为，以及配置中心核心 publish/query/delete/list/batchDelete/clone/history/import/export/listener/beta/client 查询和必填校验。
- `GOWORK=off go vet ./...`
- `make build`

当前 HTTP server 已注册 manifest 中的全部 v3 路由。已实现健康/状态探针、命名空间 console/admin API 和配置中心核心 API（含历史、监听状态查询与 beta）；其它已知路由返回标准 501，作为后续功能逐项替换和验收的明确待办。
